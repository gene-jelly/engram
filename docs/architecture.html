Engram

Persistent memory infrastructure for Claude Code

How It Works in 30 Seconds

Every time I start a session, memories from past conversations are automatically loaded. Every time you type a prompt, a hook silently searches my memory backends and injects relevant context before I see your message. I also have MCP tools to search memory on demand. The result: I remember across sessions without you needing to re-explain anything.

The key insight: Memory injection happens before Claude sees the prompt. The gap-detector hook runs in ~159ms on 80% of prompts, adding relevant observations as invisible context. Claude doesn't "decide" to remember — the system makes relevant memories available automatically.

Session Lifecycle

Session Start
  • anticipatory-loader.py pre-loads 25 observations from SQLite using time-of-day patterns, recency, and graph centrality
  • Context index plugin injects ~50 recent observations into system prompt
  • CLAUDE.md + MEMORY.md loaded as static context (policies, system facts, project state)
  • sync-skills.sh pulls 62 skill files from the vault
Every Prompt
  • gap-detector.sh fires, routes to LITE or FULL pipeline
  • Relevant observations injected as additionalContext
  • skill-activator.sh detects matching skills and loads them
  • Claude sees: static context + pre-loaded obs + gap-detected obs + user prompt
On Demand
  • Claude calls MCP tools: search, timeline, get_observations
  • Explicit memory queries when automatic injection isn't enough
  • Claude can update MEMORY.md with new persistent facts
  • Observations are written to SQLite + ChromaDB as work proceeds
Session End + Overnight
  • sync-skills.sh pushes any new/modified skills back to vault
  • All observations persisted in SQLite + ChromaDB
  • Nightly consolidation: Hebbian learning strengthens frequently-accessed observations
  • Ebbinghaus decay fades stale, unaccessed observations over time
